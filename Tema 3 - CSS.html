Tema 03 · CSSTres formas

Tres lugares donde escribir CSS

Inline, internal, external

Inline

En la etiqueta

Atributo style directamente en un elemento HTML. Solo afecta a ese elemento.

Caso puntual · evita usarlo
Internal

En el <head>

Una etiqueta <style> dentro del <head> con todas las reglas. Afecta a todo este archivo.

Pruebas y demos rápidas
External

En un .css aparte

Un archivo style.css que se enchufa con <link>. Reutilizable en muchas páginas.

La forma profesional

Tema 03 · CSS¿Cuál uso?

Recomendación práctica

Para casi todo, external.

External
El archivo .css separado del HTML, enlazado con <link>.
Tu opción por defecto. Reutilizable, ordenado, fácil de cachear.
Internal
El bloque <style> dentro del <head>.
Demos rápidas, una página única, prototipos sueltos.
Inline
El atributo style="" en un elemento concreto.
Excepciones. Sobrescribe todo lo demás (más sobre esto en la cascada).

Regla de oro: contenido en el HTML, presentación en el CSS.

Tema 03 · CSSSelectores básicos

Cómo apuntar a los elementos

Cinco selectores que cubren el 90%

h1, p, ul
Selector de elemento: por nombre de etiqueta.
Aplica a todos los <h1>, <p> o <ul> de la página.
.destacada
Selector de clase: empieza por punto.
Aplica a todos los elementos con class="destacada". El más usado.
#hero
Selector de id: empieza por almohadilla.
Aplica al único elemento con id="hero". Solo uno por página.
[type="email"]
Selector de atributo: corchetes con nombre y valor.
Aplica a todos los elementos cuyo atributo coincida. Útil en formularios.
*
Selector universal: el asterisco.
Aplica a todos los elementos. Típico para reset: * { margin: 0; }.

Tema 03 · CSSCombinar selectores

Selectores que se enchufan unos con otros

Cuatro maneras de combinarlos

Agrupar h1, h2, h3 Coma → "todos a la vez". Aplica el mismo estilo a varios selectores distintos.
Descendiente article p Espacio → "<p> dentro de <article>", a cualquier nivel de profundidad.
Hijo directo article > p ">" → solo <p> que sea hijo directo de <article>, no nietos.
Encadenar .btn.primary Sin espacio → el elemento debe tener las dos clases: btn Y primary.

Combinando estos cuatro mecanismos puedes apuntar a casi cualquier rincón de la página sin tocar el HTML.

Tema 03 · CSSUnidades

px, pt, em, rem · cuándo usar cada una

Las unidades de tamaño

px
Píxeles. Tamaño absoluto en la pantalla.
Es lo más fácil. Útil para bordes, separaciones pequeñas y casos puntuales.
pt
Puntos tipográficos. Pensados para impresión.
Mejor evítalo en web. Reservado para CSS de print.
em
Relativo al tamaño del padre. 2em = el doble que su padre.
Útil dentro de componentes. Cuidado: se acumula al anidar.
rem
Relativo a la raíz (<html>). 1rem ≈ 16px por defecto.
El más recomendable. Cambias el tamaño base y escala toda la página.

Por defecto en este curso: rem para tipografía, px para detalles finos.

Tema 03 · CSSLa cascada

Cuando dos reglas se contradicen, ¿cuál gana?

La cascada en cuatro pasos

1 · Importance
!important rompe la jerarquía.
Última opción. Si lo necesitas mucho, normalmente es síntoma de un mal selector.
2 · Specificity
Selectores más específicos ganan a los más generales.
id > class > elemento. Lo vemos en la siguiente diapositiva.
3 · Type
El origen del estilo importa.
Inline (style="") > internal (<style>) > external (.css) > navegador.
4 · Order
A igualdad de especificidad, gana la última que se haya escrito.
Por eso el orden de los archivos .css en el <head> importa.

Tema 03 · CSSEl patrón clave

El truco que vas a usar mil veces

Anclar con absolute dentro de relative

style.cssindex.html
/* el padre marca el origen */ .caja { position: relative; width: 480px; height: 280px; background: #1A1814; } /* el hijo se ancla al padre */ .punto { position: absolute; top: 100px; left: 200px; width: 80px; height: 80px; background: #C84A1F; }
<!-- los dos divs que pinta el render --> <div class="caja"> <div class="punto"></div> </div> /* uno fuera (.caja) y otro dentro (.punto) el padre se hace relative, el hijo se hace absolute, y el navegador hace el resto. */
Vista en el navegadorlocalhost
top: 100px
left: 200px
.caja → position: relative

absolute busca el ancestro más cercano que sea relative, absolute o fixed. Si ninguno lo es, se ancla a la página entera. Esa es la trampa más común: pones absolute y se va al rincón de la página porque su padre no tenía position.

Tema 03 · CSSborder-radius

Doblar las esquinas, de poco a totalmente redondo

border-radius: del cuadrado al círculo

0
Esquinas en pico. El valor por defecto.
12px
Esquinas suaves. El típico botón.
40px
Bastante redondeado. Estilo icono de app.
50%
Círculo perfecto, si ancho = alto.

Receta para un círculo: width y height iguales + border-radius: 50%. Si el ancho y el alto no son iguales, sale un óvalo.
