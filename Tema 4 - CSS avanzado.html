Tema 04 · CSS avanzadoBlock / inline

Antes de pensar en layout, entiende el flujo

Cada elemento es bloque o en línea

display: block
Ocupa toda la línea. Rompe abajo y arriba.
div, section, article…
h1, h2, p, ul, li…
header, nav, footer…
display: inline
Sigue el texto. No acepta width ni height.
El precio era 19,90 € pero ahora está a mitad y la verdad es que vale la pena por la calidad.
display: inline-block
En línea pero acepta width, height y padding.
PEGI 12 −30% PC Indie Acción Oferta

display: none lo borra del flujo (deja de ocupar). visibility: hidden lo oculta pero sigue ocupando su sitio.

Aplicaciones web · SMR17

Tema 04 · CSS avanzadoFlex vs Grid

Las dos herramientas modernas de layout — no compiten, se complementan

Flex o Grid: ¿1D o 2D?

Flexbox

Una dimensión: una fila o una columna.

Lo bueno en:

  • Navbars y barras
  • Listas alineadas
  • Botones agrupados
  • Tarjetas que se ponen una al lado de otra
  • Centrar cualquier cosa

Grid

Dos dimensiones: filas Y columnas a la vez.

Lo bueno en:

  • Layout completo de página
  • Galerías de imágenes
  • Dashboards con widgets
  • Tablas visuales
  • Composiciones tipo bento (estilo Mondrian)

En una web real conviven: Grid reparte la página entera, Flex alinea cosas dentro de cada cajón.

Tema 04 · CSS avanzadoMedia queries

La sintaxis que hace que el CSS cambie según el ancho

@media y los breakpoints

min-width: 640px
Estos estilos se aplican cuando la ventana mide 640px o más.
Móvil grande. A partir de aquí, ya hay espacio para dos columnas.
min-width: 768px
Tablet en vertical.
Aparece la navbar completa, las cards en grid de 2.
min-width: 1024px
Tablet en horizontal o laptop pequeño.
Grid de 3 columnas, sidebar visible.
min-width: 1280px
Desktop estándar.
Layout completo, espacios generosos.
max-width: 640px
Solo móvil pequeño (al revés).
Útil para esconder cosas que sólo molestan en pantallas mini.

No memorices los números. La regla: empieza pequeño y abre cuando el diseño empiece a verse mal. Eso es el breakpoint.

Frameworks de CSS.

Aplicaciones web · SMR27
</>
Tema 04 · CSS avanzadoFramework · concepto

Una caja de herramientas que ya viene montada

Un framework es CSS hecho que tú reutilizas

Sin framework

Tú escribes todo el CSS desde cero: colores, espacios, botones, tarjetas, grid, breakpoints… Toma su tiempo, pero el resultado es 100% tuyo.

Como hacerte un mueble cortando tú la madera.

Con framework

Alguien ya escribió un sistema completo de clases (.btn, .row, .col-6…). Tú solo las usas. Tu HTML se llena de clases, pero el CSS lo aporta el framework.

Como montar un mueble de IKEA: las piezas ya vienen cortadas.

Ni uno ni otro es "mejor". Cada uno sirve para una situación. Veamos las tres familias más usadas.

Aplicaciones web · SMR28
</>
Tema 04 · CSS avanzadoBootstrap · Tailwind · Material

Tres maneras muy distintas de pensar el mismo problema

Tres frameworks, tres filosofías

B

Bootstrap

Componentes

Te da una caja con piezas ya montadas: navbar, modal, carrusel, formulario. Pones la clase .btn .btn-primary y aparece un botón "estilo Bootstrap". Lleva desde 2011, lo usa media web.

Cuándo: prototipo rápido, panel de administración, MVP de cliente, "que funcione y se vea decente ya".

T

Tailwind

Utility-first

No te da componentes — te da clases mini: una para el padding, otra para el color, otra para el flex. Tu HTML se llena de p-4 bg-red-500 flex items-center. Es como construir con piezas de Lego.

Cuándo: diseño 100% propio, no quieres parecer "uno más de Bootstrap", control absoluto del estilo final.

M

Material

Design system

El sistema de diseño de Google. No es solo CSS: es una guía completa con reglas estrictas de espacios, sombras, animaciones, jerarquía. Lo ves en Android, en Gmail, en YouTube.

Cuándo: app tipo Android, producto corporativo, todo el equipo tiene que seguir las mismas reglas.

Aplicaciones web · SMR29

Tema 04 · CSS avanzado¿Cuál elijo?

Tu situación decide, no el "más moderno"

¿Qué framework para qué proyecto?

Bootstrap
Hago una web para un cliente que necesita estar online el viernes.
Componentes listos, documentación masiva, la IA lo conoce a la perfección. El 90% de plantillas de WordPress lo usan.
Tailwind
Quiero que mi web no parezca "una más" — necesito un diseño propio.
Más curva al principio, pero el resultado es totalmente tuyo. Es lo que usan webs como Vercel, Netflix Tudum o GitHub.
Material
Hago una app interna de la empresa, parecida a Gmail o Google Calendar.
Reglas estrictas, todo el equipo respeta la misma guía. Perfecto si el usuario final ya está acostumbrado a apps de Google.
Sin framework
Es una landing pequeña, una página personal, un proyecto de clase.
Aprendes más, no metes 200 KB de CSS que no usas, controlas cada píxel. Recomendado para este curso.

Tema 04 · CSS avanzadoReto de valor

Reto · 45 min · mismo brief para toda la clase

La hero section de Tomorrowland 2026

Diseña la primera pantalla (la que se ve sin scroll) de la web del festival más internacional del mundo. La IA escribirá el código. Tu trabajo es decidir qué se muestra, qué se calla y por qué.

Brief común
Tomorrowland 2026 · festival de música electrónica · dos fines de semana: 17–19 y 24–26 julio · Boom, Bélgica · +18 años · 400.000 personas de más de 200 países · 800 artistas en 15 escenarios · 2 cabezas de cartel sin desvelar
Tipos de entrada
Full Madness Pass (3 días) 350 € · Day Pass 135 € · Global Journey con vuelo + Dreamville desde 850 €
Objetivo de la web
Dar a conocer el evento y vender entradas a todo el planeta.
Entrega
HTML + CSS + 3 frases defendiendo tus decisiones · 45 min
El proceso

4 pasos · en este orden

  1. Piensa ¿Quién va a abrir esta web y desde qué dispositivo?
  2. Decide ¿Qué tiene que entender en 5 segundos? ¿Qué quieres que haga después?
  3. Pide Constrúyelo con la IA. Flex + grid + media queries. En móvil reorganiza, no encojas.
  4. Defiende 3 frases: qué priorizaste, qué dejaste fuera, por qué.
→ El código bonito sin pensamiento no aprueba. Lo importante es la conversación con la IA y la defensa de las decisiones.
