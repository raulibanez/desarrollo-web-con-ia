Tema 01 · HTMLEtiquetas sin cierre

Sintaxis · void elements

No todas las etiquetas se cierran.

Sin cierre — void elements

No tienen contenido. Una sola etiqueta y listo.

  • <br>salto de línea
  • <hr>línea horizontal divisoria
  • <img>imagen
  • <input>campo de formulario
  • <meta>metadato (dentro del head)
  • <link>enlace a CSS o recursos

+ source · track · area · col · embed · wbr

Cierre opcional — pero ciérralas

El navegador las entiende sin cierre, pero el código se vuelve ilegible.

  • <li>dentro de una lista
  • <p>antes de otro bloque
  • <tr> <td> <th>celdas de tabla

Cierra siempre estas etiquetas aunque puedas no hacerlo. Más legible para ti y para la IA.

Antes, en XHTML se escribía <br/>. Ahora, en HTML5, basta con <br> — las dos formas son válidas.

Tema 01 · HTMLCuándo usar cada cosa

Más importante que la sintaxis · cuándo se usa

¿Cuándo uso cada nivel de título?

<h1>
El título principal de la página.
Uno por página. Es lo más importante de toda la pantalla.
<h2>
Las secciones grandes en las que se divide la página.
"Sobre mí", "Servicios", "Contacto", "Preguntas frecuentes".
<h3>
Subsecciones dentro de una sección.
Dentro de "Servicios": "Diseño", "Desarrollo", "Consultoría".
<p>
Un párrafo de texto. El contenido en sí.
Todo lo que sea texto continuo, salvo títulos y elementos de lista.

Tema 01 · HTMLCuándo usar listas

Sí, lo de los puntitos · pero ¿para qué de verdad?

Una lista está en todas partes.

Navegación
Los enlaces del menú superior son una lista.
Inicio · Productos · Precios · Contacto.
Features
"Qué incluye este plan" en una página de pricing.
Cada beneficio o característica es un <li>.
Pasos
"Cómo funciona" o tutoriales numerados.
Aquí sí importa el orden → <ol>.
FAQ
Las preguntas frecuentes son una colección.
Y cada respuesta puede tener su propia mini-lista dentro.

Tema 01 · HTMLEl mapa semántico

HTML semántico · cada zona tiene su etiqueta

Una página real está hecha de zonas con nombre.

header · logo + navegación
nav · enlaces principales
main > section (hero) · titular + CTA
article · contenido
aside · barra lateral
footer · créditos y enlaces
  • <header>La cabecera. Suele llevar logo + menú.
  • <nav>El menú de navegación principal.
  • <main>El contenido único de esta página.
  • <section>Una zona temática dentro del contenido.
  • <article>Una pieza independiente: post, noticia, ficha.
  • <aside>Contenido secundario o complementario.
  • <footer>El pie. Avisos, enlaces, créditos.

El idioma de la web · si lo nombras, la IA lo construye

Vocabulario que tienes que reconocer.

Sección de bienvenidaHeroLa primera pantalla: titular grande + llamada a la acción.

Tema 01 · HTMLAtributos clave

Atributos que vas a usar todo el rato

Cinco atributos que basta con saber.

class
Ponerle un "tipo" a un elemento para darle estilo después con CSS.
class="boton-principal", class="card destacada".
id
Identificador único en la página. Solo uno con cada id.
id="formulario-contacto". Útil para enlazar y para JS.
href
A dónde lleva un enlace.
href="/precios", href="https://ejemplo.com".
src
De dónde sale la imagen (o el vídeo, el script…).
src="logo.png", src="/img/hero.jpg".
alt
Descripción de una imagen para quien no la pueda ver.
Accesibilidad y SEO. Nunca lo dejes vacío en imágenes con contenido.

Tema 01 · HTMLAnatomía de un prompt

Cómo se compone una buena petición

Un prompt útil tiene cuatro partes.

CONTEXTOEstoy aprendiendo HTML. Quiero una página estática, sin frameworks.
OBJETIVOHazme una página "Sobre mí" con un hero, una sección de proyectos y un footer.
RESTRICCIONESSolo HTML (sin CSS aún). Usa etiquetas semánticas. Que sea válido y comentado.
FORMATODame el archivo completo en un solo bloque de código.

Las cuatro partes no son obligatorias siempre, pero recordarlas evita el 90% de los "no era esto lo que quería".

Tema 01 · HTMLAntes / después

El mismo objetivo, dos resultados distintos

Antes & después de un prompt.

— Prompt vago

"Hazme una web sobre mí."

La IA tiene que inventar todo: estructura, tono, contenido, secciones. El resultado raramente coincide con lo que querías.
+ Prompt accionable

"Hazme un archivo HTML llamado index.html con: un <header> con mi nombre, una <section> hero con un titular y un subtítulo, una <section> de proyectos con 3 cards (título + descripción), y un <footer> con un enlace a mi correo."

Sabes el vocabulario (header, hero, cards, footer) → la IA traduce, tú revisas. Ese es el oficio nuevo.
