Concepto

Un componente es una
pieza de UI reutilizable.

Un botón, un campo de texto, un menú lateral. Tiene un nombre, unos estados (normal, hover, error…) y una forma estándar de usarse. Combinando piezas se construye cualquier producto.

ReutilizableEl mismo botón aparece en 200 sitios — y siempre se ve y se comporta igual.
EstandarTienen nombres aceptados — toast, drawer, chip. Hablar el vocabulario.
PredecibleEl usuario ya sabe usar un menú hamburguesa. No reinventes la rueda.
CombinableUna página es un puzzle de componentes encajados — no se diseña pixel a pixel.

Tema 02 · UI · FundamentosEstados
28 / 34Cada componente tiene varios

Fundamentos · 01

Estados.
El mismo botón, siete vidas.

Un componente no es una imagen — es una máquina con varios estados. Diseña los siete, no solo el primero.

Default El estado de reposo. Cómo lo ves al cargar la página.
Hover El ratón está encima. Confirma "esto es clicable".
Focus Llegado con Tab. Imprescindible para accesibilidad.
Active El dedo o ratón está presionando. Dura un instante.
Disabled No se puede pulsar todavía — falta rellenar algo.
Loading Procesando la acción. Bloquea la doble pulsación.
Error Algo falló al pulsar. Suele combinarse con un toast.
